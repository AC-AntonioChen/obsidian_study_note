## MySQL 基础架构
### 说说 MySQL 的架构？
Mysql 架构上由client层，server层，存储引擎层构成。其中server层的组件有连接器，查询缓存，分析器，优化器，执行器，
client是mysql对接外部的客户端，比如mysql提供的jdbc。用于接受外部的数据库请求。
连接器会校验请求的合法性，如果一个查询请求会将请求转发到查询缓存层，如果查询缓存命中则直接返回，如果查询缓存没有则转发给分析器。
查询缓存在Mysql8.0之后就被移除了，缓存不是数据库的强项。
分析器会对sql进行词法分析，这一步的目的是校验sql语句是否合法，如果有语法错误，会报错。如果语法ok，则将请求下发到优化器。
优化器会挑选最适合该sql的执行计划，下发给执行器。
执行器会将sql下发给存储引擎，如果用户没有访问的表或者执行的操作的权限，执行器会报错。
存储引擎就是我们常说的myisam和innodb，不同的存储引擎有不同的数据库底层结构架构
