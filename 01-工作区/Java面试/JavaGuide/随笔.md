## 1、random.nextInt的工作原理
`Random.nextInt(int n)`方法生成的随机数是均匀分布的，这是因为它的实现方式确保了每个数字在指定的范围内被选中的概率是相等的。下面是一个简化的演示过程：
以random.nextInt(10)举例
1. `Random.nextInt(int n)`首先会生成一个几乎均匀分布在0（包含）和1（不包含）之间的浮点数。这是通过内部的**线性同余生成器**或者其他类似的算法完成的。
```
线性同余生成器是一种用来生成随机数的算法。它的工作原理是使用一个公式，这个公式需要三个参数：种子（seed）、乘数（multiplier）和增量（increment）。这个公式如下：

Xn+1​=(aXn​+c)modm

其中，Xn​是当前的随机数（或者说是种子），a是乘数，c是增量，m是模数，Xn+1​是下一个随机数。

这个公式的结果是一个新的数字，这个数字在0到m之间。然后，这个新的数字就成为下一次计算的种子。

为了得到一个0到1之间的浮点数，我们只需要把这个数字除以m就可以了。因为这个数字是在0到m之间的，所以除以m之后的结果就是在0到1之间的。
```

线性同余生成器是一种用来生成随机数的算法。它的工作原理是使用一个公式，这个公式需要三个参数：种子（seed）、乘数（multiplier）和增量（increment）。这个公式如下：


Xn+1​=(aXn​+c)modm

其中，Xn​是当前的随机数（或者说是种子），a是乘数，c是增量，m是模数，Xn+1​是下一个随机数。

这个公式的结果是一个新的数字，这个数字在0到m之间。然后，这个新的数字就成为下一次计算的种子。

为了得到一个0到1之间的浮点数，我们只需要把这个数字除以m就可以了。因为这个数字是在0到m之间的，所以除以m之后的结果就是在0到1之间的。
1. 然后，这个浮点数会被乘以`n`（在你的例子中是10），结果是一个0到`n`的浮点数。
    
3. 最后，这个浮点数会被强制转换（取整）为一个整数。因为原始的浮点数是均匀分布的，所以结果的整数也是均匀分布的。
    
这就是为什么`Random.nextInt(10)`能够生成一个0到9之间的随机数，且每个数字被生成的概率是相等的。
